Key Insight
The top help cluster (formula language confusion, 162 records) reflects that Sigma's calculated-field syntax is opaque for non-technical audit and compliance professionals. This is compounded by the permission matrix (61 records), where customers cannot determine which role/team combination grants the correct access tier. Together these two clusters represent 73% of all help-seeking in the module, pointing to a fundamental onboarding and documentation gap that likely drives unnecessary support volume and contributes to the negative NPS.

Narrative Patterns
Competitor benchmarking: Power BI, Tableau, and Workiva are cited repeatedly as active alternatives. Customers frequently reference "custom Power BI dashboards that worked better before the migration." Sigma's replacement of Power BI-based dashboards is a recurring sore point in renewal calls.
BI/API integration requests: Appears in 197 calls, often framed as "we still have to export to Power BI because Sigma doesn't meet our needs." An analytics database connector is being positioned as a solution in some CS calls but is not universally communicated.
Executive reporting use case: Audit committee reporting, board presentations, and KPI/KRI dashboards come up in nearly every demo and CS call. Customers need presentation-ready outputs, not just data tables.
Cross-module visibility gap: The request to consolidate risks, issues, controls, and audits into a single dashboard appears in 762 calls. This is the single most common theme in Gong data for this module.

Priority Recommendations
1
Fix the Sigma migration fallout
Address the three core Sigma-era regression complaints in sequence: (a) data freshness/latency: dashboards refreshing in hours not seconds is a breaking experience for executive reporting, (b) eliminate the hidden save/update workflow confusion with auto-save or clear commit affordances, and (c) restore or formally sunset ABI dashboard migration paths with proactive customer communication and migration support.
270 latency complaints + 479 customization/UX complaints + 13 NPS detractor comments at score ≤ 6
2
Build a native unified cross-module dashboard
Create a pre-built, configurable "Connected Risk" or "GRC Overview" dashboard that joins risks, issues, controls, and audit status in a single view. This is the #1 improvement request by volume (808 Gong calls, 808 tagged improvement records).
1,567 combined records across Gong and taxonomy
3
Implement row-level / audience-specific permissions
Add granular view/row-level permissions so admins can publish a single dashboard with filtered views per audience (e.g., executive sees only their entity; auditor sees full scope). This is both the #2 complaint (426 records) and the #3 improvement request (535 records).
961 combined records
4
Launch a custom views / data flexibility program
Enable the #1 improvement request: custom dashboard data views (1,138 records) by expanding the data model available to Sigma, adding more joinable fields, and providing guided templates for the most common audit/risk reporting use cases. Pair with onboarding documentation and an in-product formula wizard to close the 162-record formula language confusion gap.
1,300 combined records
5
Formalize BI/API integration as a first-class capability
Power BI and Tableau appear in 197 Gong calls and 217 improvement records as active requests. Customers who export to Power BI rather than using native dashboards represent an adoption gap and a churn risk (as demonstrated by the Workiva evaluation mention). Document and actively promote the analytics database connector; consider a co-build program with major BI tools.
414 combined records
